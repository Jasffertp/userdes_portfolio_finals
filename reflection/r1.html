R1: MY FIRST FEW WEEKS OF USERDES CLASS

Week 1:

The time when class was about to start and then miss suddenly started the class with glee and energy which sets up the mood for the whole class. It really showed me what this class is going to be, even though the subject is in siesta time miss really wants the class to be interactive and energetic but most importantly have fun while learning something in the subject. The whole session was about us just introducing ourselves and knowing our favorite apps where miss subtly inserted some UI/UX references. Miss pointed out while I’m describing my app that I was actually addressing the pain points of the app which I used to describe my app in the activity. Designing was very complicated for me, I know how to design and I was hoping I could use the design of my back then however I wasn’t able to find my code. I tried creating a website by copying a complex template and it took me about 2 days and I scrapped the project and decided to just create a simple website and it only took me half the day to achieve it, I was really glad that I decided to drop the first website because the second website was better and simple. I really liked the fact that I can saved my code in github although it made me sad when I realized that I can save the other projects that I created back then instead of saving the files somewhere and forgetting where it was saved. I’m glad that I learned how to store codes and projects in github now I’ll be able to save all the projects that I created while improving my profile.

Week 2:

The class started with us trying to figure out how to finish the userinyerface website, it was simple enough and the background was very bright for my eyes and it really made the experience of trying to finish the riddle worse. In the end it set up for what the class is about to be, we discussed about the difference between UI and UX, and what we would feel if we were to try to be a user of our website. I think the main takeaway for me was the importance of empathy towards the users, we should always try to be thoughtful about how we’re creating and designing our website because they are the people who’re going to be the users of our websites they’re going to be the one who’ll be trying the website, if they’re not happy or they have a crappy experience then most likely the website might fail or have less users. In the session we also did a kahoot I was lucky enough to get the first place in the first kahoot, I was surprised that I was even able to achieve that. The kahoot was about the UX and UI which was the topic for the day. We did a kahoot again for the second time and I made a mistake in the middle, sadly I wasn’t able to defend my first place. We also discussed about the creating personas for our website and how they’re a big factor of how our website was going to be, I was so stressed when I was trying to create a persona because I thought that the final website that we did was going to be based around those three people. Luckily my classmate said to not worry because what will be attached is the prototype of our website so I just created personas of people I think I’d get along with.

Week 3:

The previous meeting was about design. Before starting the class, we had an individual kahoot. I thought that we were going to be doing it as a class so I was worried about already starting so I closed the kahoot however, I found out that it was an individual kahoot so I answered again. Miss opened the class by asking us what design is and I learned that design was not only about the aesthetics of the object but the overall capability of the object. It really makes sense because design is basically the function of the object, what it’s “designed” to do. To follow that miss presented to the class a website and asked us what’s the problem with that website, I thought that the problem was about the design being too close to each other however, when my classmates answered it was made clear that it was not that. I really learned a lot about the user experience and user design in the website and the session gave me more ideas when creating a website and personas for my website We learned about the design life cycle and I think that it should be practiced by beginner designers to just keep on designing and constantly make adjustments if you think there should be any. I think that the design life cycle is key on designing a website because it helps it get better as time goes.


Overall I learned a lot and how I adjust in designing a website because back then I would try to create a website for myself, not considering the users of the website so I think that the sessions really levelled up the way I create websites.
